# Emoji Data Authoring

Read before editing `src/overrides.tsv` or anything in `data/`.

## The overlay is the only hand-edited surface

`src/overrides.tsv` is where you add/curate emoji. Upstream `data/` is generated by
`make fetch` Рђћ never hand-edit it. Columns (tab-separated, trailing optional):

```
emoji  action  category  order  taigi_keywords  zh_Hant_keywords  en_keywords  name  notes
```

- `patch` Рђћ add keywords to an emoji already in upstream. Build fails if it is not.
- `add` Рђћ add a brand-new emoji not in upstream (needs `category` + `name`; `version` is
  recorded as `custom`). Use sparingly Рђћ prefer upstream.
- `exclude` Рђћ drop an upstream emoji.
- Keywords are `|`-separated (terms may contain commas).

After editing: `make build` then `make test`. The drift-guard test fails if committed
`dist/emoji.json` is stale.

## Taigi keywords Рђћ authoritative-source-only

Taiwanese romanization is **never invented**. This mirrors the sibling `taigikeyboard`
repo's Core Principle #3.

- For `taigi_keywords`, prefer Т╝бтГЌ (unambiguous) and only add уЙЁждгтГЌ (canonical TL) you have
  verified against `../taigikeyboard/knowledge/taigi-phonetics-reference.md` and the
  `taigi-converter` submodule there.
- Do NOT guess tone marks or spelling. Leave уЙЁждгтГЌ out rather than ship a wrong reading.
- A Taiwanese word's identity is the **(Т╝бтГЌ, уЙЁждгтГЌ) pair** Рђћ same Hanji, different reading =
  different morpheme. Keep both readings when both are meaningful search terms.

## Versioning

`MAX_EMOJI_VERSION` (in `scripts/generate.py`) and the `Makefile` `CLDR_TAG` are pinned
together and documented in `data/SOURCES.md`. Raising the pin is a deliberate change:
`make fetch` Рєњ bump both Рєњ `make build` Рєњ review diff. Release scope/timing is user-gated.
